Client-Server Architecture

1. What is Client-Server Architecture?

Answer

Client-Server Architecture is a computing model in which a client requests services or resources, and a server processes those requests and returns responses.

Client → Request → Server
Client ← Response ← Server

The client is usually responsible for the user interface, while the server handles business logic and data processing.

2. What is a Client?

Answer

A client is a device or application that requests services from a server.

  • Example: Web Browser
  • Example: Mobile App
  • Example: Desktop Application

The client initiates communication.

3. What is a Server?

Answer

A server is a system that receives requests from clients, processes them, and returns responses.

  • Responsibility: Execute business logic
  • Responsibility: Authenticate users
  • Responsibility: Access databases
  • Responsibility: Serve resources

4. Why do we use Client-Server Architecture?

Answer

Benefits include:

  • Benefit: Separation of concerns
  • Benefit: Better scalability
  • Benefit: Easier maintenance
  • Benefit: Centralized data management
  • Benefit: Improved security
Follow-up

Interviewer:

What is meant by separation of concerns?

Answer

Different responsibilities are divided between components.

  • Client: Handles presentation/UI
  • Server: Handles business logic
  • Database: Handles data storage

This makes applications easier to develop and maintain.

5. What protocol is commonly used between client and server?

Answer

The most common protocols are:

  • Protocol: HTTP
  • Protocol: HTTPS

HTTPS is preferred because it encrypts communication.

Follow-up

Interviewer:

What is the difference between HTTP and HTTPS?

Answer
HTTP HTTPS
Not encrypted Encrypted
Less secure More secure
Uses Port 80 Uses Port 443
Vulnerable to interception Protects transmitted data

6. What is a Request?

Answer

A request is a message sent by the client asking the server to perform an action. 

GET /users

The client is requesting user information.

7. What is a Response?

Answer

A response is the data returned by the server after processing a request. 

{
  "id": 1,
  "name": "John"
}

8. What are the advantages of Client-Server Architecture?

Answer
  • Advantage: Centralized management
  • Advantage: Better security
  • Advantage: Easy updates
  • Advantage: Resource sharing
  • Advantage: Scalability

9. What are the disadvantages of Client-Server Architecture?

Answer
  • Disadvantage: Server can become a single point of failure
  • Disadvantage: Higher infrastructure cost
  • Disadvantage: Network dependency
  • Disadvantage: Server overload under heavy traffic

10. Can multiple clients connect to one server?

Answer

Yes. A single server can handle requests from many clients simultaneously.

  • Example: Thousands of web browsers accessing a website
  • Example: Multiple mobile apps connecting to the same backend
Follow-up

Interviewer:

How does a server handle multiple clients at the same time?

Answer
  • Technique: Multithreading
  • Technique: Asynchronous processing
  • Technique: Event-driven architectures
  • Technique: Load balancing

This allows concurrent request handling.

11. What is a Stateless Server?

Answer

A stateless server does not store client session information between requests.

Each request must contain all required information.

  • Benefit: Better scalability
  • Benefit: Easier load balancing
  • Benefit: Reduced server memory usage
Follow-up

Interviewer:

Why are stateless systems easier to scale?

Answer

Because any server instance can process any request without relying on previous interactions. Requests are independent.

12. What is a Three-Tier Architecture?

Answer

A three-tier architecture divides an application into:

Presentation Layer (UI)
↓
Business Logic Layer
↓
Database Layer
  • Layer: Presentation Layer (UI)
  • Layer: Business Logic Layer
  • Layer: Database Layer

This improves modularity and maintainability.

13. What is the difference between Two-Tier and Three-Tier Architecture?

Answer
Two-Tier Three-Tier
Client ↔ Database Client ↔ Server ↔ Database

Three-tier architecture is more secure and scalable because clients do not directly access the database.

Common Interview Trap

Interviewer:

Can a client directly access a database?

Answer

Technically yes, but it is generally not recommended.

Reasons:

  • Reason: Security risks
  • Reason: Business logic bypass
  • Reason: Difficult access control
  • Reason: Tight coupling

Preferred Approach:

Client → Server → Database
