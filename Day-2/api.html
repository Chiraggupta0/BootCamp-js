APIs, DNS, Hosting & Frontend-Backend Workflow

1. APIs

Easy

Q: What is an API?

Answer

API stands for Application Programming Interface. It is a set of rules that allows different software applications to communicate with each other.

For example, when a frontend application needs user data, it does not access the database directly. Instead, it sends a request to an API, and the API processes the request and returns the required data.

In simple terms, an API acts as a messenger between two systems, taking requests from one system and delivering responses from another.

Follow-up: Why are APIs important?

  • Benefit: Enable communication between different systems
  • Benefit: Promote code reusability
  • Benefit: Hide internal implementation details
  • Benefit: Allow frontend and backend teams to work independently
  • Benefit: Make applications scalable and maintainable

Medium

Q: What is a REST API?

Answer

REST stands for Representational State Transfer. A REST API follows REST principles and uses HTTP methods such as GET, POST, PUT, PATCH, and DELETE to perform operations on resources. 

GET /users
POST /users
PUT /users/1
DELETE /users/1

REST APIs are widely used because they are lightweight, scalable, and easy to understand.

Follow-up: What is a resource in REST?

A resource is any piece of data that can be accessed through an API.

  • Example: /users
  • Example: /products
  • Example: /orders
  • Example: /employees

Each resource is identified by a unique URL.

Hard

Q: Explain the difference between REST and SOAP.

Answer

REST SOAP
Lightweight Protocol-based
Uses JSON or XML Uses XML only
Faster More standardized
Easier to implement Supports transactions
Uses HTTP methods directly More complex

REST is generally preferred for modern web and mobile applications, while SOAP is often used in enterprise systems where strict security and reliability are required.

Follow-up: When would you choose SOAP over REST?

  • Use Case: High security requirements
  • Use Case: Transaction management
  • Use Case: Formal service contracts
  • Use Case: Banking and financial systems

2. DNS

Easy

Q: What is DNS?

Answer

DNS stands for Domain Name System. Its primary role is to convert human-readable domain names into machine-readable IP addresses.

google.com ↓ 142.250.x.x

Since humans remember names better than numbers, DNS makes internet usage much easier.

Follow-up: Why can't we use IP addresses directly?

Although we can use IP addresses directly, they are difficult to remember. DNS provides a user-friendly naming system that maps domain names to IP addresses.

Medium

Q: What happens when you type a website URL into a browser?

Answer

  1. Browser checks its cache.
  2. Operating system checks DNS cache.
  3. DNS lookup is performed.
  4. DNS returns the server's IP address.
  5. Browser establishes a connection with the server.
  6. HTTP/HTTPS request is sent.
  7. Server processes the request.
  8. Response is returned.
  9. Browser renders the webpage.

This entire process usually happens within milliseconds.

Follow-up: What is DNS caching?

DNS caching stores previously resolved domain names temporarily. This reduces lookup time and improves performance.

Hard

Q: Explain recursive and iterative DNS lookups.

Answer

Recursive Lookup Client ↓ DNS Resolver ↓ Finds Final Answer ↓ Returns IP
Iterative Lookup Client ↓ DNS Server 1 ↓ DNS Server 2 ↓ DNS Server 3 ↓ Final IP

Follow-up: Which lookup is commonly performed by clients?

Clients usually perform recursive lookups because the DNS resolver handles the complexity.

3. Hosting

Easy

Q: What is Hosting?

Answer

Hosting is the process of deploying and storing an application on a server so users can access it over the internet.

Without hosting, users would not be able to access a website or application from anywhere in the world.

Follow-up: What is a Server?

A server is a computer or software system that receives requests from clients, processes them, and returns responses.

Medium

Q: Difference between Domain and Hosting?

Answer

Domain: google.com, amazon.com

Hosting: The server where the application runs.

Domain = House Address Hosting = Actual House

Follow-up: Can a website work without a domain?

Yes. A website can be accessed directly using its IP address.

Hard

Q: Types of Hosting?

Answer

  • Shared Hosting: Multiple websites share one server.
  • VPS Hosting: Virtualized server with better control.
  • Dedicated Hosting: Entire server dedicated to one customer.
  • Cloud Hosting: Resources distributed across multiple servers.

Follow-up: Why is Cloud Hosting Popular?

  • Advantage: Scalability
  • Advantage: High availability
  • Advantage: Fault tolerance
  • Advantage: Pay-as-you-go pricing
  • Advantage: Better reliability

4. Frontend & Backend Workflow

Easy

Q: What is Frontend and Backend?

Answer

Frontend is the part of an application that users interact with directly.

  • Frontend Technology: HTML
  • Frontend Technology: CSS
  • Frontend Technology: JavaScript
  • Frontend Technology: React

Backend is responsible for business logic, authentication, APIs, and database interactions.

  • Backend Technology: Spring Boot
  • Backend Technology: Node.js
  • Backend Technology: Django

Follow-up: Can a website exist without a backend?

Yes. Static websites built with HTML, CSS, and JavaScript can work without a backend.

Medium

Q: How do frontend and backend communicate?

Answer

Frontend ↓ HTTP Request ↓ Backend API ↓ Database ↓ Backend Response ↓ Frontend UI Update

The backend usually sends data in JSON format. 

{
  "name": "John"
}

Follow-up: Why is JSON commonly used?

  • Reason: Lightweight
  • Reason: Human-readable
  • Reason: Easy to parse
  • Reason: Supported by most programming languages

Hard

Q: Explain the complete frontend-backend workflow when a user logs in.

Answer

User ↓ Frontend ↓ HTTP Request ↓ Backend ↓ Database ↓ Backend ↓ HTTP Response ↓ Frontend ↓ User

Follow-up: Why doesn't the frontend directly access the database?

  • Reason: Security
  • Reason: Business logic protection
  • Reason: Validation
  • Reason: Access control

5. Most Important Interview Question

Q: Explain the complete journey of a request from the browser to the server and back.

Answer

User ↓ Browser ↓ DNS Lookup ↓ Server ↓ Application Logic ↓ Database ↓ Application Logic ↓ Server Response ↓ Browser Rendering ↓ User
  1. Browser checks cache.
  2. DNS resolves domain name into an IP address.
  3. Browser establishes a TCP connection.
  4. SSL/TLS handshake occurs for HTTPS.
  5. HTTP request is sent.
  6. Server receives request.
  7. Business logic executes.
  8. Database operations occur.
  9. Response is generated.
  10. Browser renders the response.

This answer demonstrates understanding of APIs, DNS, networking, frontend-backend communication, hosting, and the complete request lifecycle, which interviewers commonly expect from full-stack candidates.
